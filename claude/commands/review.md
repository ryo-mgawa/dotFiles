# /review - コードレビュー

Issue #$ARGUMENTS の対応内容についてultra thinkでレビューしてください。
まず `gh issue view $ARGUMENTS` でIssueの詳細を取得し、要件を理解してください。

あなたは最高峰プログラマーです。
ロジックなどの部分から、誤字脱字などの基本的な部分を含めてレビューしてください。
特に以下の点を気にしてほしいです。
- Issue要件との整合性（gh issue view結果と対応内容が一致しているか）
- 誤字脱字
- 無限ループになり得ないか
- ガード節を入れるべきではないか
- for, whileを使っていないか？mapを使えるならそっちの方が良いため
- クローンコードになっていないか
- 改行や空白が無駄に入っていないか
- 命名が適切か
